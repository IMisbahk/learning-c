# Pivot

This project originally started with the plan of learning C through a traditional book. (ritchie's C programming)
the book explains concepts well, but the pace is extremely slow for the way i prefer to learn. 
but i learn faster by reading concise explanations and immediately implementing the ideas in code.

because of this, I decided to pivot away from the book and move toward faster online resources and documentation while continuing to build programs inside this repository.

The goal of the project has not changed.

The goal is to understand C deeply by writing code, experimenting with concepts, and observing how the language behaves at a low level.

## New Approach

Instead of progressing chapter by chapter through a single book, the learning process will now focus on:

- reading short explanations from online resources and documentation
- implementing the concept immediately in code
- experimenting and modifying programs to observe behavior
- writing notes while learning

## Why This Pivot

The objective of this repository is not to complete a book.
the objective is to understand C well enough to use it for systems-level programming and to build real programs.
if a different learning path achieves that goal more effectively, the change is justified.
