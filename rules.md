# Project Rules

This repository follows a few simple rules.

## Learning Approach

read concept
implement it immediately
experiment with it
break it
understand why it broke

## Code Style

i will be using camelCase for variables and functions

Minimal comments
Only comment when something important needs explanation

## Compilation

Programs are compiled with:

cc file.c -o program
./program

(using a .vscode/tasks.json build task)

later will replace it with a Makefile.

## Philosophy

The goal is not to finish a resource.
the goal is to understand how C actually works.
